# 项目事实（仅背景，无评价，请自行验证是否属实）
- 定位：多租户文件/图片托管 SaaS，按卡密开通，面向付费客户。
- 技术栈：Cloudflare Workers（单个 worker）；D1（SQLite，元数据）；R2（文件）；Cloudflare Images（图片）；Durable Object（限流/暴破锁）；Cloudflare for SaaS 自定义域。
- 代码：全部逻辑在 D:\系统默认\桌面\畅密\tuku\index.js；同目录还有 wrangler.jsonc、schema.sql、migration-v2.sql。仓库 jxgcr/tuku，通过 GitHub Actions 手动部署。
- 前端：服务端渲染。客户端页面 PAGE_HTML、运营台页面 ADMIN_HTML（路径 /scfw），都是 worker 内的反引号模板字符串，在返回时被求值。
- 鉴权：客户用「卡号+密码」登录/开通，服务端签发会话令牌；运营台用 ADMIN_KEY。卡有效性由外部「畅密」系统 /api/verify 校验。
- 域名：客户端 link.aistela.com；worker 直连 tuku.5209696.xyz；运营台在 /scfw。
- 存储分三处：图片走 Cloudflare Images，其它文件走 R2，元数据/账户/相册在 D1。
- 技术注意（事实，非评价）：因前端是反引号模板字符串、下发时才求值，源码里的 \/ \n \" 等转义会被处理；审前端 JS 时，请以「下发后实际执行」的 <script> 内容为准，而非直接读源码。

# 审查维度与检查项（逐条核）
A. 逻辑与架构合理性
- 数据模型（customers/images/albums 等）字段、约束、外键、NOT NULL 是否自洽；三处存储（D1/R2/Images）之间的一致性如何保证；有无孤儿数据（删一处漏删另一处）。
- 请求路由、状态流转、边界条件、并发与竞态、幂等性（重复提交/重试会不会双写或错算）。
- 容量/配额的计算口径是否准确、能否被绕过。

B. 稳定性与可靠性
- 错误处理与失败模式：某一后端（D1/R2/Images/畅密）超时或报错时的行为；部分失败是否留下不一致状态。
- 上传链路（普通/大文件分片/断点续传/重试）在中断、网络抖动、并发下的正确性。
- Worker 平台限制（CPU 时间、内存、子请求数、请求体大小）下的表现；冷启动；大列表/大文件的极端情况。

C. 安全性
- 认证与授权：会话令牌的签发/校验/过期/防篡改；权限判定是否只依赖服务端可信身份。
- 多租户隔离：每一个读/写/删/改是否严格按当前身份作用域，能否越权访问他人数据（含直链、枚举、IDOR）。
- 密码存储、暴破防护、密钥管理（是否入库/入 git/回显）。
- 注入与内容安全：SQL 注入、XSS（含上传可执行内容如 HTML/SVG 被内联渲染）、SSRF、CSRF、开放重定向。
- 直链/公开资源的访问控制模型；限流；管理后台暴露面；输入校验与文件类型处理。
- 合规与账号策略风险（如是否依赖违反服务条款的做法）。

D. 数据、备份与成本
- 迁移脚本是否安全可重放；有无备份/恢复方案；哪些操作会造成不可逆数据丢失。
- 存储/操作/流量的计费模型与扩展性；成本随规模的变化。

E. 客户使用便捷性
- 开通/登录流程的步数与可理解性；错误提示是否人话、可操作。
- 上传体验（拖拽/粘贴/多选/进度/大文件/失败重试）、文件管理（选择/批量/重命名/详情/搜索/排序/相册）。
- 空状态、加载与感知性能、移动端（导航/网格/触屏操作/弹层）、无障碍（对比度/焦点/键盘/aria）、文案与国际化。

F. 页面视觉与布局
- 视觉层级、留白与信息密度、组件一致性、宽屏空间利用、响应式与移动端断点、暗色模式、对比度与可读性、图标/间距/圆角等细节一致性、整体专业度。请在桌面宽屏（如 1900px）与手机（如 375px）两种宽度下分别检视。

G. 代码可维护性
- 单文件 + 模板字符串内嵌前端的组织方式带来的可读性/可测性问题；重复、耦合、命名、注释与实现是否一致；改动某处的回归风险。

H. 运维与部署
- CI/回滚/密钥注入/迁移执行方式；可观测性（日志、监控、告警）；出问题时的定位与恢复能力。
